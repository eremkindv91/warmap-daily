# WarMap Daily

WarMap Daily показывает не одну «правильную» линию, а проверяемую историю решений: кто заявил об изменении, какие независимые данные его подтверждают, кто одобрил геометрию и что изменилось со вчерашнего дня.

Публичный сайт: <https://eremkindv91.github.io/warmap-daily/>

Карта публикует геолокированные точечные обновления из разрешённых источников с задержкой не менее 24 часов. Репозиторий не содержит синтетических фронтовых событий и не копирует чужую полигональную геометрию без лицензии.

## Что уже работает

- отдельные слои контроля, оспариваемых зон, изменений, событий и предыдущей границы;
- diff за 24 часа, 7 и 30 дней;
- evidence bundle и матрица подтверждения для каждого изменения;
- раздельный реестр российских, украинских и независимых источников;
- поиск населённых пунктов по вариантам названий RU/UK/EN;
- каноническая карточка населённого пункта;
- переключатель RU/UK/EN для основной навигации и поиска;
- режим экономии трафика без базовых тайлов и вторичных событийных слоёв;
- архив снимков, SHA-256 и журнал публикаций;
- автоматическая проверка доступности источников;
- ежедневный сбор координат из прямых ссылок в официальных постах DeepStateUA;
- GitHub Pages и fail-closed CI/CD.

## Локальный запуск

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python -m http.server 8000
```

Откройте <http://localhost:8000>.

## Проверки

```bash
.venv/bin/python -m unittest discover -s pipeline/tests -v
.venv/bin/python pipeline/validate_snapshot.py \
  data/current.geojson \
  --evidence data/evidence.json \
  --sources data/sources.json
.venv/bin/python pipeline/validate_data.py
.venv/bin/python pipeline/validate_schemas.py
node --check assets/app.js
```

Проверки отклоняют:

- confidence ниже 0,93;
- публикацию раньше 24 часов;
- пустую или невалидную геометрию;
- пересекающиеся полигоны;
- аномальную площадь;
- неизвестные source ID и evidence ID;
- геометрию без ручного одобрения, reviewer и review timestamp.

## Публикация снимка

1. Добавьте источники в `data/sources.json`.
2. Создайте записи доказательств в `data/evidence.json`.
3. Подготовьте `candidate.geojson` по `schemas/snapshot.schema.json`.
4. Проверьте кандидата командами выше.
5. Опубликуйте только после ручной проверки:

```bash
.venv/bin/python pipeline/publish_snapshot.py candidate.geojson --approved
```

Команда:

- сохранит прежний снимок;
- рассчитает географический diff и площадь;
- определит изменившиеся статусы населённых пунктов;
- соберёт поисковый индекс;
- создаст датированный архив;
- обновит manifest и status;
- добавит запись с SHA-256 в audit log.

LLM и feed-адаптеры не рисуют линию контроля. `pipeline/ingest_feeds.py` сохраняет только документы в очереди проверки.

## Публичные статические endpoints

- `data/status.json` — состояние публикации;
- `data/current.geojson` — текущая геометрия;
- `data/previous.geojson` — предыдущая версия;
- `data/changes.geojson` — последний diff;
- `data/updates.geojson` — геолокированные точечные обновления;
- `data/reference-control.geojson` — закрашенная датированная оценка контроля РФ из открытой карты Wikimedia Commons;
- `data/events.json` — проверенные события;
- `data/claims.json` — заявления сторон;
- `data/evidence.json` — доказательства;
- `data/settlements-index.json` — канонический индекс топонимов;
- `data/snapshots/index.json` — архивный manifest;
- `data/audit-log.json` — журнал публикаций.

## Источники и лицензии

Официальное заявление подтверждает факт заявления, а не территориальный контроль. Для DeepStateUA публикуются только координаты из прямых ссылок в публичных постах, с явной атрибуцией и статусом оценки одного OSINT-источника. API DeepStateMAP и полигональная геометрия не копируются. Геометрия ISW не используется: сервис указывает, что геоданные нельзя копировать без письменного согласия. OpenStreetMap применяется только как базовая карта с обязательной атрибуцией ODbL.

Закрашенный справочный слой `data/reference-control.geojson` векторизован из файла [2022 Russian invasion of Ukraine.svg](https://commons.wikimedia.org/wiki/File:2022_Russian_invasion_of_Ukraine.svg), автор Viewsridge, лицензия [CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/). В интерфейсе явно указана дата исходной геометрии — 24 апреля 2026 года. Слой отделён от подтверждённых snapshots и не выдаётся за оперативную границу на текущую дату. Привязка использует опубликованные координатные границы базовой карты Wikimedia: 21,5–40,7° в. д. и 44,1–52,7° с. ш.

Карта публикуется с задержкой и не предназначена для навигации, эвакуации или решений, связанных с безопасностью.
