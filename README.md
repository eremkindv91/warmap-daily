# WarMap Daily

Статический MVP ежедневной карты изменений территориального контроля по проверяемым открытым источникам. Стартовый набор данных намеренно пуст: проект не публикует синтетическую геометрию или вымышленные события.

## Быстрый запуск

```bash
cd warmap-daily
python3 -m http.server 8000
```

Откройте `http://localhost:8000`.

## Публикация данных

1. Сформируйте кандидат `candidate.geojson` по схеме `data/current.geojson`.
2. Для каждого объекта укажите `id`, `status`, `valid_from`, `confidence`, `evidence_ids` и `moderation`.
3. Проверьте файл:

```bash
python3 pipeline/validate_snapshot.py candidate.geojson
```

4. Только после ручной проверки опубликуйте:

```bash
python3 pipeline/publish_snapshot.py candidate.geojson --approved
```

Публикация сохранит прежний снимок, создаст датированный архив и обновит статус сайта. Порог confidence для публичной геометрии — 0,93; минимальная задержка — 24 часа.

## Важное ограничение

Workflow ежедневно проверяет целостность и публикует уже одобренные файлы, но не создаёт геометрию автоматически. Адаптеры источников выключены до отдельной проверки лицензий, условий использования и устойчивости официальных endpoints.

Нельзя копировать геометрию ISW без письменного согласия: сам сервис указывает, что геоданные являются его интеллектуальной собственностью. Для базовой карты используется OpenStreetMap с обязательной атрибуцией.

## Структура

- `index.html`, `assets/` — интерфейс;
- `data/` — текущий, предыдущий снимки, diff, события и источники;
- `pipeline/` — fail-closed валидация и публикация;
- `config/sources.yml` — реестр будущих адаптеров;
- `.github/workflows/pages.yml` — тесты и GitHub Pages.

Карта является аналитической оценкой, отображается с задержкой и не предназначена для навигации, эвакуации или решений, связанных с безопасностью.
